# PromptProxy アーキテクチャ設計書

## 1. 概要
本アプリは、CLIツール「Aider」とブラウザ上のAI（Gemini/Claude等）を中継するTauri(Rust) + Reactアプリである。Aiderからは「OpenAI API互換のローカルサーバー」として振る舞い、受け取ったリクエストを人間が手動でブラウザにコピペしやすい形式に再構築する。

## 2. 状態遷移 (State)
アプリは以下の3つの状態を持つ。
1. **Idle (待機)**: AiderからのHTTPリクエストを待機している状態。
2. **Pending (入力待ち)**: Aiderからリクエストを受け取り、ブラウザAIの応答（コード修正）を人間がテキストエリアに入力するのを待っている状態。
3. **Resolving (応答完了)**: 人間が入力したテキストをJSONにラップしてAiderにHTTPレスポンスとして返し、Idleに戻る状態。

## 3. データ処理フロー（ハイブリッド方式）
Aiderから `POST /v1/chat/completions` にJSONが送られてきた際、Rust側で以下の処理を行う。

1. **JSONの解体と分離**: `messages` 配列の中から「システムルール・ユーザーの指示」と「ソースコードのコンテキスト」を分離する。
2. **XMLファイルの生成**: ソースコードのコンテキスト部分のエスケープを解除し、`<repository>` などのタグで囲んだXML形式の文字列を作成。アプリのローカルディレクトリに `context.xml` として保存する。
3. **プロンプトの生成**: システムルールとユーザーの指示を結合し、末尾に「【重要】出力は挨拶や解説を一切省き、SEARCH/REPLACEブロックのみを使用すること」という厳格なルールを付与したプレーンテキストを生成する。
4. **フロントエンドへの送信**: `context.xml` の保存先パスと、生成したプロンプトのテキストをTauriのイベントでReact側に送信し、HTTPリクエストは `oneshot` チャネルで保留（await）する。
